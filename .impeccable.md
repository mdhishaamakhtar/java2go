## Design Context

### Users

Experienced Java developers (mid-to-senior level) transitioning to Go. Comfortable with OOP, the JVM ecosystem, and Java verbosity. They visit as a reference — not for entertainment — and want clarity and efficient knowledge transfer. Likely reading on a desktop in a focused, possibly dim environment.

### Brand Personality

Educational, clear, calm.

### Aesthetic Direction

Deep space dark theme. Dual-accent identity: **amber (`#c9b85a`) = Java**, **cyan (`#4ec9b0`) = Go**. This color language is the core visual concept — amber represents the familiar, cyan represents the destination. Blue (`#4a7aff`) is the neutral accent for UI chrome and links.

**Theme:** Dark — chosen for the developer reading context (code-heavy, terminal-adjacent, focused sessions).

**Anti-references:** Bright/playful developer marketing sites, glassmorphism, neon-on-dark, maximalist dashboards, anything that competes visually with the code examples.

### Design Principles

1. **Dual-accent identity is the concept.** Amber = Java, cyan = Go. Use consistently across all surfaces, including social images and the README banner. Never swap the colors.

2. **Educational clarity over flair.** Every decorative element must earn its place. The interface must not compete with the code examples — those are the product.

3. **Deep space baseline.** `#080812` is darker than typical dark themes. It creates a terminal-like focus environment. Don't lighten it for "accessibility theatre" — keep contrast ratios honest using the existing token system.

4. **Typographic weight contrast is the primary hierarchy tool.** Bold headings, regular body, monospace code. Three distinct registers — don't add more.

5. **Calm and restrained motion.** No animations unless they serve comprehension (e.g., smooth scroll). The site's authority comes from stillness, not kinetics.
